## 把[击水飞舟](https://blog.ai2me.qzz.io/)网站从0到1搭建个人博客说透
作为一名在职场摸爬滚打多年的技术人，“搭建个人博客”这个想法在我心里搁置了足足三年。曾经改造过一款开源Blog系统，却因服务器续费、域名备案等一堆琐事半途而废(很多也由于必须是免费开源的执念)；想记录职场踩坑经验、分享技术干货，甚至尝试做点副业，却一直找不到趁手的工具。直到2025年底刷到博主分享Hexo迁移Astro Paper的实战，我才终于下定决心：2026年，必须把这个“迟到”的博客落地。

用5W1H框架梳理整个搭建过程，既有技术细节的沉淀，也有从0到1的思考，希望能给同样想拥有个人博客的你带来启发。

## What：搭建的是什么样的博客？
我最终落地的是一个**基于Astro框架+Astro Paper模板二次开发**的多功能个人博客。它不只是简单的文章发布平台，更承载了三重核心定位：
- 职场日记本：记录后端开发中的技术难点、项目复盘、职场成长感悟；
- 技术分享站：拆解编程技巧、框架实战、工具使用心得，主打“实用可复制”；
- 副业试验田：后续计划上线付费专栏、技术咨询入口，探索内容变现可能。

博客支持Markdown/MDX全格式解析、深色/浅色模式自动切换、静态搜索、RSS订阅，还优化了中文排版和SEO配置，既保证了开发效率，又兼顾了读者体验。

## Why：为什么偏偏选择Astro+Astro Paper？
选择比努力更重要，尤其是技术选型阶段。我放弃了曾经熟悉的Hexo、WordPress，也pass了热门的Next.js，最终锁定Astro，核心原因有三点：

1. **性能碾压传统博客框架**：Astro的“岛屿架构”太香了——默认将页面渲染为纯静态HTML，只在需要交互的组件（比如搜索框）加载JS，首屏加载速度比Hexo快3倍以上，Lighthouse评分轻松破95；
2. **开发门槛低，灵活性高**：无需学习新语法，支持React、Vue等组件无缝嵌入，既可以像Hexo一样快速生成静态页，又能灵活定制功能，完美平衡“易用性”和“扩展性”；
3. **生态成熟，模板丰富**：经过两年发展，Astro的社区模板已经非常完善。我最初花2天从零搭建了简易博客，后来发现GitHub上的成熟模板能节省80%的开发时间，最终选择了Astro Paper——它轻量无冗余，支持内容分类、标签系统、OG图自动生成，比astro-theme-cactus的功能更贴合个人博客场景，又比astroship（偏营销网站）、astrofy（偏作品集）更聚焦内容创作。

当然，还有个关键原因：解决了我当年半途而废的痛点——Astro博客支持静态部署，无需购买昂贵服务器，Netlify、Vercel均可免费托管，域名备案也简化了很多。

## When：整个搭建过程花了多久？
从想法落地到博客上线，前后共耗时**10天**，时间分配非常清晰：
- 第1-2天：技术调研。对比Hexo、Jekyll、Astro等框架的性能、生态、学习成本，最终敲定Astro；
- 第3-4天：从零试错。跟着Astro官方文档搭建简易博客，熟悉文件结构、路由规则、Content Collections内容管理方案；
- 第5-6天：模板选型与改造。对比5款热门Astro模板（包括astro-theme-cactus、astroship等），最终fork Astro Paper并完成基础汉化；
- 第7-9天：二次开发。定制导航栏、优化中文排版、集成搜索功能、配置SEO和RSS订阅，AI工具帮我快速解决了样式兼容、语法报错等问题；
- 第10天：部署上线。购买域名、配置Netlify自动部署、测试跨浏览器兼容性，最终成功上线。

## Where：博客部署在哪里？内容存放在哪里？
### 部署平台：Netlify（免费版足够用）
选择Netlify而非Vercel、GitHub Pages，主要是因为：
- 与Astro生态兼容性极佳，支持自动部署（推送GitHub代码后自动构建）；
- 免费版提供HTTPS证书、自定义域名绑定、CDN加速，国内访问速度稳定；
- 内置表单功能，后续可轻松添加“留言板”“技术咨询”模块，无需额外配置后端。

### 内容存储：双备份方案
- 核心内容：以Markdown/MDX文件形式存储在GitHub仓库，通过Astro的Content Collections进行管理，支持标签分类、草稿状态控制；
- 静态资源：图片、图标等存放在Netlify的CDN，同时备份到阿里云OSS，避免丢失；
- 数据备份：GitHub仓库开启分支保护，重要更新前手动备份，确保内容安全。

## Who：这个博客是为谁而建？
- 核心读者：刚入行的技术新人（分享入门技巧）、同行开发者（交流技术难点）、想搭建博客的朋友（输出实战教程）；
- 潜在读者：对副业变现、职场成长感兴趣的职场人；
- 最终服务：我自己。通过“输出倒逼输入”，梳理职场经验，沉淀技术知识，同时尝试内容变现，把个人影响力转化为实际价值。

## How：搭建过程中的关键技术细节与避坑指南？
### 1. 模板改造的核心步骤
Astro Paper的二次开发并不复杂，关键在于“保留核心功能，剔除冗余代码”：
- 汉化与定制：修改`src/config.ts`中的网站信息，替换`src/components`中的导航栏、页脚等组件的英文文本，优化中文排版（调整字体、行间距）；
- 功能增强：集成Pagefind静态搜索（比模板自带的搜索更精准），添加RSS订阅文件（`src/pages/rss.xml.ts`），配置OG图自动生成（利用Astro的Satori集成）；
- 样式优化：通过TailwindCSS自定义主题色，修改`src/styles/globals.css`调整页面布局，确保移动端适配。

### 2. AI工具的神助攻
作为非专职前端开发者，很多样式兼容、语法问题让我头疼，AI帮我节省了大量时间：
- 快速定位报错：将TypeScript、Astro配置报错复制给AI，秒出解决方案，比如解决“Pagefind搜索索引生成失败”“MDX组件引入报错”等问题；
- 样式定制：描述需求（比如“让导航栏在滚动时变色”“优化中文列表间距”），AI直接生成TailwindCSS代码，无需手动查文档；
- 批量处理：用AI批量转换Markdown文章的格式，自动补全Frontmatter字段，提高内容迁移效率。

### 3. 避坑指南（新手必看）
- 不要盲目从零搭建：除非需要高度定制化功能，否则直接使用成熟模板，比如Astro Paper、astro-theme-cactus，避免重复造轮子；
- 注意Node版本兼容：Astro v5要求Node.js 18+，部分模板（如astro-theme-cactus）已升级到Node.js 22，提前确认版本要求，避免依赖安装失败；
- 部署前必做测试：运行`pnpm build && pnpm preview`本地测试构建结果，重点检查动态路由（比如文章详情页、标签页）是否正常访问；
- 域名配置：如果使用国内访问，建议选择阿里云、腾讯云的域名，备案流程更顺畅，避免因未备案导致访问受限。
